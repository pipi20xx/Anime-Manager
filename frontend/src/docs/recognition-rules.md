# 自定义识别词 (Custom Rules)

**核心处理文件**: `backend/recognition_engine/title_cleaner.py` (逻辑位于 `pre_clean` 函数)

此环节拥有双重功能：既可以作为 **“橡皮擦”** 清除无效词汇（屏蔽），也可以作为 **“记号笔”** 提前强制标记正确的 ID 或季数（提取）。

---

## 🎯 实战演练 (Case Study)

假设文件名为：
`[SumiSora][Fate Stay Night Unlimited Blade Works][25v2][BDrip][1080p][HEVC_FLAC].mkv`

以下展示 7 种不同维度的规则写法及其效果：

| 模式 | 写法 (规则) | 效果 / 解释 |
| :--- | :--- | :--- |
| **1. 简单屏蔽 (Block)** | `SumiSora` | **删除** "SumiSora"。<br>本质是 `SumiSora =>` (替换为空) 的简写。 |
| **2. 简单替换 (Replace)** | `Unlimited Blade Works => UBW` | 将长标题缩写为 **"UBW"**，利于后续搜索。 |
| **3. 正则提取元数据 (Regex Meta)** | `Fate Stay Night => {[tmdbid=60553]}` | 识别到该词时，直接 **强制锁定 TMDB ID**，跳过云端搜索。 |
| **4. 定位器偏移 (Locator Offset)** | `[ <> v2] >> +100` | 定位 `[` 和 `v2]` 之间的数字 (25)，执行 +100。<br>结果：**125** |
| **5. 捕获组计算 (Group Formula)** | `\[(\d+)v2\] => {[e=\1@*2+1]}` | 捕获数字 (25)，代入公式 `25 * 2 + 1`。<br>结果：强制集数为 **E51** |
| **6. 组合规则 (Combined)** | `SumiSora && Unlimited Blade Works => UBW` | 从左到右依次执行：先删除 `SumiSora`，再将 UBW 替换。 |
| **7. 多字段强制注入** | `Fate Stay Night => {[tmdbid=60553;s=1;type=tv]}` | 同时强制锁定多个字段：TMDB ID、季数、媒体类型。 |

## 🛠️ 强制提取字段详解 (Metadata Tags)

在替换目标中使用 `{[key=value]}` 语法，可跳过后续识别逻辑，直接锁定属性。

| 字段 | 说明 | 示例 |
| :--- | :--- | :--- |
| **`tmdbid`** | **(最常用)** 强制指定 TMDB ID，跳过搜索，直接精准匹配。 | `{[tmdbid=80000]}` |
| **`type`** | 强制指定资源类型。仅支持 `tv` 或 `movie`。 | `{[type=movie]}` (强制识别为电影) |
| **`s`** | 强制指定季度 (Season)。 | `{[s=1]}` |
| **`e`** | 强制指定集数 (Episode)。支持公式计算。 | `{[e=12]}` 或 `{[e=\1@+1]}` |

## ⚡ 正则与语法提示 (Regex Guide)

系统基于 **Python `regex` 模块** 实现，支持标准正则表达式和增强特性。

### 1. 核心语法特性
*   **忽略大小写**: 系统默认开启了 `re.I` (Ignore Case) 标志，因此 `hevc` 也能匹配 `HEVC`。
*   **边界匹配**: `\bWord\b` (全词匹配，防止误伤 `Word2`)
*   **模糊匹配**: `Word.*Test` (匹配中间任意字符)
*   **位置锚点**: `^Start` (仅匹配开头), `End$` (仅匹配结尾)
*   **字符集**: `[a-z]` (字母), `\d+` (数字)
*   **CJK边界保护**: 系统使用智能边界判定，支持中文/日文边界检测，防止误伤标题的一部分。

### 2. 规则类型本质
*   **简单替换 (A => B)**: 左边是正则，右边是普通字符串（支持 `\1` 引用捕获组）。
    *   示例: `Part\s*(\d+) => Pt.\1` (将 "Part 01" 替换为 "Pt.01")
*   **简单屏蔽**: 本质上就是 `正则 => 空字符串` 的简写。
    *   示例: `\d+bit` (屏蔽 "10bit", "8bit" 等)
*   **组合规则 (&&)**: 支持用 `&&` 连接多条规则，按顺序依次执行。
    *   示例: `SumiSora && Unlimited Blade Works => UBW` (先删除 SumiSora，再替换标题)
*   **强制元数据提取**: 使用 `{[key=value]}` 语法，支持多字段组合。
    *   示例: `Fate => {[tmdbid=60553;s=1;type=tv]}` (同时锁定多个字段)

### 3. 集数计算公式支持
集数计算公式使用 `@` 符号标识，支持多种运算风格：

| 公式写法 | 说明 | 示例 (基于捕获组 \1 = 25) |
| :--- | :--- | :--- |
| `@+1` | 加法 | `25 + 1 = 26` |
| `@*2+1` | 复合运算 | `25 * 2 + 1 = 51` |
| `@EP+1` | EP前缀格式 | `EP25 + 1 = EP26` |

**完整示例：**
```
\[(\d+)v2\] => {[e=\1@*2+1]}
```
- 捕获组 `\1` 匹配到数字 `25`
- 公式 `@*2+1` 表示：`25 * 2 + 1 = 51`
- 最终强制集数为 **E51**

### 4. 路径鲁棒性增强
当启用"强制单文件模式"时，系统会自动进行路径处理：
- 将 `/` 和 `\` 替换为 `_`，防止路径分隔符被误解析
- 针对复杂路径自动提取文件名部分进行规则匹配

**示例：**
原始路径: `/path/to/[SumiSora] Title - 01.mkv`
处理后: `_path_to_[SumiSora] Title - 01.mkv`

### 5. 智能清洗特性
系统在预处理阶段会自动执行以下清洗：
- **装饰符号清除**: 自动清除 `★`, `☆`, `■`, `◆`, `●` 等装饰性符号
- **冒号脱敏**: 将 `:` 替换为空格，防止 Anitopy 解析崩溃
- **斜杠处理**: 自动处理 `/` 分隔符，防止误判
- **名称叠加防护**: 防止因正则替换产生的名称重复 (如 "字幕组字幕组")
- **空壳括号清理**: 递归清理被掏空的括号块 (如 `[ ]`, `( )`, `【 】`)
- **不可见字符清除**: 清除零宽空格等不可见字符
