# 下载规则配置 (RSS)

**核心处理文件**: `backend/rss_core/matcher.py` (逻辑位于 `Matcher.check_match` 函数)

RSS 规则的核心是 **过滤**。系统会获取订阅源中的最新条目，然后逐一用您配置的规则进行比对。只有 **同时满足** 以下条件的资源才会被推送下载：

*   满足「包含关键词」的所有条件
*   不触发「排除关键词」中的任意条件
*   如果指定了「作用范围」，则资源必须来自指定的订阅源

---

## 1. 普通关键词模式 (Simple Mode)

适合绝大多数场景。不区分大小写。

### 包含关键词 (Must Contain)
逻辑结构：**(A 且 B) 或 (C 且 D)**

| 符号 | 含义 | 示例 |
| :--- | :--- | :--- |
| **空格** | **逻辑与 (AND)** | `海贼王 1080p` <br>标题需同时包含 "海贼王" 和 "1080p" |
| **\|** | **逻辑或 (OR)** | `1080p | 4k` <br>标题含 "1080p" 或 "4k" 均可 |
| **混用** | **组合逻辑** <br>(空格优先级高于 \|) | `海贼王 1080p | 鬼灭之刃 4k` <br>匹配 (海贼王 AND 1080p) OR (鬼灭之刃 AND 4k) |

### 排除关键词 (Must Not Contain)
即"黑名单"。一旦标题包含以下任意一个词，直接丢弃。

| 写法 | 含义 |
| :--- | :--- |
| `720p` | 只要包含 "720p" 就丢弃 |
| `720p | MP4` | 既不要 720p，也不要 MP4 格式<br>(用 `|` 分隔多个排除词) |

## 2. 正则表达式模式 (Regex Mode)

> **核心提示**：完全支持 Python 标准正则语法（默认开启 `re.I` 忽略大小写）。

此模式下，「包含关键词」和「排除关键词」框内均填写 **一条完整的正则表达式**。

### 实战示例

| 场景 | 正则表达式 | 说明 |
| :--- | :--- | :--- |
| **精确匹配集数** | `[MyGroup].*One Piece.*-\s+\d{3,4}\s+.*1080p` | 匹配组名开头，中间含 One Piece，紧接3-4位集数，最后1080p |
| **限定字幕组** | `(ANi\|Bahamut).*我推的孩子.*1080p` | 只匹配 ANi 或 Bahamut 字幕组发布的资源 |
| **忽略大小写** | `(?i)spy\s?x\s?family.*1080p` | 虽然系统默认忽略大小写，但你仍可显式指定 `(?i)` |
